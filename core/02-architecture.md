# 2. 技術アーキテクチャ

公開されている一次情報（プレスリリース）と、ソフトバンクによる技術解説（Insights記事）を
統合すると、Crystal Intelligence は以下の4つの技術要素で構成されていると理解できます。

## 2.1 基盤プラットフォーム：OpenAI「Frontier」＋ o1シリーズ

**出典**: SB OAIプレスリリース「OpenAIの法人向けAIプラットフォーム『Frontier』を基盤に」、
OpenAI公式Frontier発表（2026年2月5日）

- Crystal Intelligence は、OpenAI のエンタープライズAIエージェントプラットフォーム
  **「Frontier」** を基盤として構築されている
- 推論能力に優れる **「o1シリーズ」** など、OpenAIの最先端LLMがベースモデルとして利用される
- ポイントは、Crystal Intelligence が**独自にゼロから学習したモデル**ではなく、
  OpenAIの既存の最先端モデル・基盤を**企業導入向けに構成・統合したレイヤー**である、という点

**理解のコツ**: 「Frontier」はOpenAI側のプラットフォーム名、「Crystal Intelligence」は
それを使ってSB OAI Japanが日本企業向けに提供するサービス名、と役割を分けて覚えると混同しにくい。

Frontier自体は独立した製品として2026年2月5日にOpenAIから正式発表されており、
機能構成（Business Context / Agent Execution / Security & Governance）や対象顧客など、
比較的まとまった一次情報が公開されています。この基盤部分だけを体系的に理解したい場合は
次章 [03-frontier-platform.md](03-frontier-platform.md) で詳しく扱います。

## 2.2 エンタープライズ統合：安全なRAGと長期記憶

**出典**: ソフトバンク Insights「クリスタル・インテリジェンス：企業変革をもたらす最先端AI」

- 各企業の社内データ・基幹システムのソースコードなどを**安全に隔離された環境**で解析・統合する
- 統合されたデータは**他社に漏洩・再利用されない**仕組みが構築されている（テナント分離）
- AIが企業の「長期記憶」を持つ、という表現がされており、単発の質問応答ではなく、
  蓄積された企業固有の文脈を踏まえた継続的な支援を志向している
- ユースケース例として、複雑な基幹システムのソースコードを解析し、プログラムの置き換え
  （モダナイゼーション）を支援する、といった動作イメージが紹介されている

**技術的に整理すると**、これは一般的な生成AIエンタープライズ導入における
「RAG（Retrieval-Augmented Generation：検索拡張生成）」の考え方に、
企業ごとの継続的なコンテキスト保持（＝長期記憶）を組み合わせたものと解釈できます。
ただし、具体的なベクトルDBの種類や検索アルゴリズムなど、実装レベルの詳細は非公開です。

ソフトバンクグループはこの長期記憶の仕組みについて**特許を取得済み**と発言しており、
「企業を理解するAI」というメッセージングは2025年2月の初回発表時点から一貫して
強調されています。発言の詳細・タイムラインは [08-understanding-the-enterprise.md](08-understanding-the-enterprise.md) を参照。

## 2.3 自律型AIエージェント群の連携

**出典**: SB OAIプレスリリース、および関連する発言（X/Twitter等での言及）

- Crystal Intelligence は単一のチャットボット型インターフェースではない
- **複数のAIエージェントが協調**して、以下のような複雑な業務プロセスを自律的に遂行する
  システムとして計画されている
  - 財務関連業務
  - 文書作成
  - 顧客対応
- 「マルチエージェントオーケストレーション」に近い設計思想と考えられる（※業界一般の用語による整理）

**理解のポイント**: 従来の「1つのプロンプト→1つの回答」という単純なQ&A型ではなく、
複数の専門エージェントがタスクを分担・連携しながら、業務プロセス全体（ワークフロー）を
遂行することを目指している点が、エンタープライズ向けサービスとしての特徴です。

## 2.4 FDE（Forward Deployed Engineer）による実装支援

**出典**: SB OAIプレスリリース

- 技術ドキュメントを企業側に渡して終わり、ではなく、**OpenAIとソフトバンクのエンジニア（FDE）が
  企業へ直接常駐・伴走**する開発スタイル
- FDEが担う領域は一気通貫で以下をカバーする：
  1. データの統合（企業の基幹システム・社内データとの連携）
  2. セキュリティ・ガバナンスの設計
  3. 既存システムへの実装

**理解のポイント**: 「Forward Deployed Engineer」はPalantir社が確立したことで知られる
職種概念で、顧客企業の現場に深く入り込み、汎用プロダクトを個社の業務に合わせて
カスタマイズ実装するエンジニアを指します。Crystal Intelligenceの導入モデルも、
「プロダクトを売って終わり」ではなく「人によるコンサルティング＋実装」を伴う点が特徴です。
（Palantirにおける定義は一般的な業界知識であり、SB OAIが同一の定義を用いているとは
明言されていない点に留意してください＝※参考情報）

## 2.5 4要素の関係整理

| 要素 | 役割 | 出典の種類 |
|---|---|---|
| Frontier / o1シリーズ | 基盤となるLLM・プラットフォーム | 公式プレスリリース |
| 安全なRAG＋長期記憶 | 企業データとの統合・文脈保持 | ソフトバンクTech解説 |
| 自律型マルチエージェント | 業務プロセスの自律遂行 | 公式プレスリリース＋補足言及 |
| FDEによる実装 | 導入・運用の伴走支援（人的要素） | 公式プレスリリース |

この4つは独立した機能ではなく、「基盤モデル（2.1）」の上に「企業データ統合（2.2）」が乗り、
それを「マルチエージェント（2.3）」が業務プロセスとして活用し、
最終的に「FDE（2.4）」が現場に実装する、という**積み上げ構造**として理解すると全体像が掴みやすくなります。
